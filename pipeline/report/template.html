Methodology & Limitations

Dependency extraction uses the GitHub Dependency Graph SBOM export endpoint, which returns an SPDX JSON document. Dependencies are identified by their purl (Package URL) from the SBOM's externalRefs.

Vulnerability scanning uses the OSV.dev querybatch API, querying each dependency by purl. Results are aligned 1-to-1 with the input ordering.

OpenSSF Scorecard results come from the public REST API. If a project has not been scored (404), the value is recorded as "N/A" and the pipeline continues without error.

RepoReaper-lite is a lightweight engineering-hygiene heuristic (0–100) based on easily-checkable signals (README, LICENSE, CI, tests, docs, recent activity, stars). It is not the original GHTorrent-backed RepoReapers tool.

Limitations: SBOM completeness depends on GitHub's dependency detection for each language ecosystem. Some repos may have no SBOM or incomplete dependency graphs. OSV coverage varies by ecosystem. Scorecard availability depends on whether the project has been indexed.
